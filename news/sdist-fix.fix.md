Fix sdist completeness so `python -m build --sdist` ships the OWASP CRS rule files under `pycoraza/coreruleset/rules/` and so source installs build `libcoraza.so` locally when no manylinux wheel matches. A new sdist build hook (`hatch_build.py:CrsFetchHook`) downloads and stages CRS at the pinned `CRS_TAG` before packaging. The wheel build hook (`native/build_ffi.py`) now also fetches CRS as a safety net and invokes `native/scripts/build-libcoraza.sh` when `LIBCORAZA_PREFIX` is unset and we're not under cibuildwheel — emitting a clear "requires Go 1.25+" message when Go is missing.
