Expose the matched-rule chain on every blocked transaction. `Transaction.matched_rules()` now returns the ordered list of CRS rules that fired (`MatchedRule(id, severity, message)`); `Interruption.rule_id` is populated from the last match in the chain instead of always being `0`. The default Flask and Starlette block responses now include `rule_id`, `msg`, and `status` so operators can triage a CRS false positive without re-running in detect mode. **Behavior change**: `Interruption.rule_id` previously always returned `0` — callers that asserted on this value need to update their expectations.
