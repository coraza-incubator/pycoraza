OWASP Coraza · for Python

A web application firewall that lives inside your Python app.

pycoraza wraps the Go-based Coraza engine via libcoraza.so and exposes idiomatic middleware for Flask, FastAPI, and Starlette. Security-first defaults, the OWASP Core Rule Set bundled, one import away from blocking attacks on every request.

Why pycoraza

Secure by default

on_waf_error="block", mode=DETECT, inspect_response=False. Every default is the one you'd pick after a pentest.

Real middleware, not a proxy

Runs inside your process. Access environ / scope, write a custom on_block, skip routes with a predicate, attach metadata — all in-band.

CRS in one import

recommended(), balanced(), strict(), permissive(). The OWASP Core Rule Set with paranoia and threshold knobs exposed as keyword arguments.

Three framework adapters

Flask via WSGI. FastAPI and Starlette via ASGI, with Coraza running in a thread so it doesn't stall the event loop.

Fail-closed by construction

Adapters catch CorazaError and return 500 rather than letting a framework's exception handler silently pass the request through.

No sidecar, no Go toolchain

Linux wheels ship libcoraza.so vendored. Install with pip and you're done. Source distributions still require Go 1.25+ and SWIG 4+.

Install

Pick the extra that matches your framework.

pip install pycoraza                 # core only
pip install "pycoraza[flask]"        # + Flask middleware
pip install "pycoraza[fastapi]"      # + FastAPI / Starlette middleware
pip install "pycoraza[all]"          # everything

Quickstart

Three adapters, one API shape. Click a tab to switch.

from flask import Flask
from pycoraza import create_waf, WAFConfig, ProcessMode
from pycoraza.coreruleset import recommended
from pycoraza.flask import CorazaMiddleware

app = Flask(__name__)

waf = create_waf(WAFConfig(
    rules=recommended(paranoia=1),
    mode=ProcessMode.BLOCK,
))

app.wsgi_app = CorazaMiddleware(app.wsgi_app, waf=waf)

@app.get("/")
def index():
    return {"ok": True}
from fastapi import FastAPI
from pycoraza import create_waf, WAFConfig, ProcessMode
from pycoraza.coreruleset import recommended
from pycoraza.fastapi import CorazaMiddleware

waf = create_waf(WAFConfig(
    rules=recommended(paranoia=1),
    mode=ProcessMode.BLOCK,
))

api = FastAPI()
api.add_middleware(CorazaMiddleware, waf=waf)

@api.get("/")
async def index():
    return {"ok": True}
from starlette.applications import Starlette
from starlette.middleware import Middleware
from pycoraza import create_waf, WAFConfig, ProcessMode
from pycoraza.coreruleset import recommended
from pycoraza.starlette import CorazaMiddleware

waf = create_waf(WAFConfig(
    rules=recommended(),
    mode=ProcessMode.BLOCK,
))

app = Starlette(middleware=[Middleware(CorazaMiddleware, waf=waf)])

Now send a malicious request: 

curl -i "http://127.0.0.1:5000/?q=../../etc/passwd"
# HTTP/1.1 403 Blocked
# {"error":"blocked","rule_id":930110,...}

Secure defaults, explained

mode=ProcessMode.DETECT
Log-only out of the box. Flip to BLOCK after you've reviewed false positives in your logs. A misconfigured rule in detect mode logs noise; in block mode it takes your service down.
on_waf_error="block"
If the WAF can't evaluate a request (allocation failure, invalid SecLang at init, whatever), the adapter returns 500. Opt into "allow" only when availability matters more than security for this deployment.
inspect_response=False
Phase 3 (response headers) and phase 4 (response body) are off unless you turn them on. Enable when you have rules that look at response content; leave off otherwise.
Static-asset bypass
Images, CSS, JS, fonts, /static/, /_next/static/, /assets/, and /favicon.ico skip the WAF by default. Override with the skip= parameter on every adapter.
